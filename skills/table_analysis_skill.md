# 适用范围
- 当用户要求读取表格、提取表格数据、核对统计结果、判断表格信息是否正确时使用。
- 适用于 Excel、CSV、Markdown 表格、从网页复制来的表格文本，以及用户直接粘贴的结构化数据。

# 工作目标
1. 先识别表格结构，包括表头、行、列、单位、时间范围和筛选条件。
2. 再提取用户真正关心的字段，例如金额、数量、得分、日期、分类、状态、合计项。
3. 对提取出来的数据做校验，确认是否存在缺失值、重复行、格式异常、单位不一致或明显不合理的值。
4. 对需要汇总的数据重新计算，不直接相信表格里的合计行。
5. 最后给出清晰判断：数据是否正确、哪里可能有问题、建议如何修正。

# 分析步骤
1. 明确用户问题：用户是要提取数据、汇总数据、核对结果，还是做结论判断。
2. 确认数据来源：如果用户提供文件名，优先使用工具读取文件；如果用户直接粘贴表格，就直接解析文本。
3. 建立字段映射：把表头和用户问题中的指标对应起来，避免列名相似导致误读。
4. 执行数据校验：检查空值、重复、类型、单位、时间范围和筛选条件。
5. 重新计算关键结果：对合计、平均值、比例、排名、差异值等关键指标重新算一遍。
6. 输出结论：先给结论，再给关键证据和必要的计算过程。

# 输出要求
- 不要只复述表格内容，要明确回答“是否正确”或“关键结论是什么”。
- 如果表格数据不足以判断，要说明缺少哪些字段或条件。
- 如果发现错误，要指出具体行、列、字段或计算项。
- 如果涉及大量明细，优先用简洁列表呈现，不要输出复杂 Markdown 表格。
- 如果是金额、比例、日期等字段，要保留清晰单位和格式。

# 注意事项
- 不要凭空补全表格中不存在的数据。
- 不要直接相信表格中的总计、均值、百分比，关键数值需要重新核算。
- 如果用户的问题涉及多个筛选条件，必须逐项确认条件是否已经应用。
- 如果表格来自文件，回答中要说明本次分析基于哪个文件或哪段数据。
