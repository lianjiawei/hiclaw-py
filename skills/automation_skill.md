# 适用范围
- 当用户要求执行多步骤操作、批量处理文件、自动化重复任务时使用。
- 适用于需要编写脚本或命令来完成的任务，而不是逐步手动操作。

# 工作目标
1. 识别任务是否适合用脚本自动化。
2. 选择合适的脚本语言（优先 Bash，复杂逻辑可用 Python）。
3. 编写清晰、可执行的脚本。
4. 安全执行脚本，并反馈结果。

# 判断标准
以下情况**强烈建议使用 Bash 工具**：
- 批量文件操作（重命名、移动、复制、删除等）
- 日志分析或文本处理（grep、awk、sed 等）
- 数据统计或汇总（计算总和、平均值、计数等）
- 格式转换（CSV 转 JSON、Markdown 转 HTML 等）
- 需要循环或条件判断的重复操作
- 生成报告或汇总文件

以下情况**可以逐步操作**：
- 简单的单文件读取或写入
- 只需要回答一个问题（不需要处理文件）
- 用户明确要求逐步解释

# 执行步骤
1. **分析任务**：确认任务是否涉及多步骤或批量操作。
2. **选择工具**：
   - 简单命令：直接写 Bash 一行命令
   - 复杂逻辑：先写脚本文件（`.sh` 或 `.py`），再执行
3. **编写脚本**：
   - 使用清晰的变量名和注释
   - 添加错误处理（如文件不存在时的提示）
   - 优先使用当前环境可用的命令
4. **执行脚本**：
   - 先用 `echo` 或 `cat` 预览脚本内容
   - 确认无误后再执行
   - 捕获并展示执行结果
5. **反馈结果**：
   - 总结执行结果
   - 如有错误，说明原因并建议修正

# 输出要求
- 执行前简要说明将要做什么
- 执行后给出清晰的结果总结
- 如果涉及文件修改，说明修改了哪些文件
- 如果执行失败，提供错误信息和可能的解决方案

# 注意事项
- 不要执行可能破坏系统的危险命令（如 `rm -rf /`）
- 批量删除操作前，先列出将要删除的文件让用户确认
- 当前环境优先使用 `python` 而不是 `python3`
- 不保证安装了 `gh` 等额外命令行工具，不要默认依赖它们
- 如果任务涉及敏感操作（删除、覆盖），先说明影响范围
